# 5-way comparison: openblas / parallel / eplinalg — kind10 (REAL/COMPLEX(KIND=10))

- Source: `reports/cmp5/cmp5.tsv` (3120 (routine,key,size) rows over 75 routines)
- Five variants: `openblas-omp1` and `openblas-omp4` (epblas-openblas overlay), `parallel-omp1` and `parallel-omp4` (epblas-parallel overlay), and `eplinalg-omp1` (Fortran reference, serial baseline).
- All four overlay binaries link the SAME `tests/epblas-parallel/perf/target_kind10/perf_<r>.c` source — only the C-overlay symbol differs.
- Same `BLAS_PERF_{ITERS,WARMUP,INCX,INCY}=200/20/1/1`; per-routine default sizes; pinned via `taskset` (P-cores 0 or 0..3).
- `eplinalg-omp1` = migrated_GFs from the `parallel-omp1` run; `mig_*` columns are sanity readings of the same migrated `_serial` symbol from each of the four runs (expected to be ~equal since `_serial` contains no OpenMP).

**Units: all columns are GF/s (throughput) — larger is better.** A `parallel/openblas` GF/s ratio **> 1.0** means parallel is faster; the firm bar is par ≥ ob in every cell. (The separate interleaved per-routine harness reports a par/ob *cycle* ratio instead, where smaller is better and < 1.0 = par faster — the reciprocal view. See `doc/optimization-findings.md` → "Reporting convention".)

> ⚠️ **Near-parity cells here are a SCREEN, not a verdict — do not read a sub-1.0 ratio as a regression.** This cmp5 is single-shot (one timing per cell, L3 at `iters=10`) and NOT interleaved, so it carries block-drift bias plus, at OMP=4 with small N, thread-pool start-up noise. Any par/ob ratio in roughly the **0.90–1.10 band is inside the noise** and can swing several percent run-to-run. This applies to BOTH the rewritten L3 routines (`esymm` `esyrk` `esyr2k` `etrsm` `etrmm` `ysymm` `yhemm` `yherk` `yher2k` …) AND near-parity L1/L2 routines — e.g. **`yher2` reads ~0.92× here yet is at/above parity under the interleaved harness** (U 0.957–0.977, L ~1.00 cycle ratio); likewise treat sub-1.0 single-shot reads for `edot`, `etpmv`, `etbsv`, `yaxpy`, `yher` as unverified, not as losses. Verdicts come ONLY from the **interleaved min-of-15 cycle harness** (par/ob alternated run-by-run to cancel block drift; `perf stat -e cycles:u`, pinned), read against a measured par-vs-par noise floor (~1.00 at OMP=1, ~1.004 at OMP=4). The five recently-fixed serial routines (`eynrm2` `erot` `yerot` `eaxpy` `yher2`) and the rewritten L3 set are all signed off there. See memory `project_x87_accumulator_spill` and `project_l3_fused_packer_pattern` for the per-routine interleaved results and commits.
