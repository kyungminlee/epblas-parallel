# 5-way comparison: openblas / parallel / eplinalg — kind10 (REAL/COMPLEX(KIND=10))

- Source: `reports/cmp5/cmp5.tsv` (3524 (routine,key,size) rows over 75 routines)
- Five variants: `openblas-omp1` and `openblas-omp4` (epblas-openblas overlay), `parallel-omp1` and `parallel-omp4` (epblas-parallel overlay), and `eplinalg-omp1` (Fortran reference, serial baseline).
- All four overlay binaries link the SAME `tests/epblas-parallel/perf/target_kind10/perf_<r>.c` source — only the C-overlay symbol differs.
- Same `BLAS_PERF_{ITERS,WARMUP,INCX,INCY}=200/20/1/1`; per-routine default sizes; pinned via `taskset` (P-cores 0 or 0..3).
- `eplinalg-omp1` = migrated_GFs from the `parallel-omp1` run; `mig_*` columns are sanity readings of the same migrated `_serial` symbol from each of the four runs (expected to be ~equal since `_serial` contains no OpenMP).

> ⚠️ **L3 parity is NOT signed off from this single-shot sweep.** This cmp5 is single-shot (one timing per cell, L3 at `iters=10`) and NOT interleaved, so it carries block-drift bias plus, at OMP=4 with small N, thread-pool start-up noise. For the rewritten kind10 L3 routines (`esymm`, `esyrk`, `esyr2k`, `etrsm`, `etrmm`) individual cells can swing several percent either way from run to run — read the per-routine medians, not single cells, and do not read a sub-1.0 L3 cell here as a regression. Parity for those routines is signed off separately with an **interleaved min-of-15 cycle harness** (par/ob alternated run-by-run to cancel block drift; `perf stat -e cycles:u`, pinned), against a measured par-vs-par noise floor (~1.004 at OMP=4, ~1.00 at OMP=1). Under that harness all five sit at parity — e.g. esymm OMP=1 reads 0.993–1.016 across the full SIDE×UPLO×{64..512} grid. See memory `project_l3_fused_packer_pattern` for the per-routine interleaved results and commits.
